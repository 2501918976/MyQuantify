
## 应用设置需求文档 (纯文本版)

### 1. 基础功能设置

#### 1.1 多语言切换

应用需要提供一个设置项，允许用户在 **简体中文** 和 **英文** 之间切换界面的显示语言。切换应尽量立即生效。

#### 1.2 开机自启动

提供一个布尔开关，用于控制应用程序是否在 Windows 操作系统启动时自动运行。

#### 1.3 每日数据周期定义

这是一个重要的时间配置。用户可以设置一天的数据统计的起始时间。

-   **目的：** 确保用户的活动数据（如专注时长和效率分析）能够精确地跨越午夜，匹配用户实际的工作日周期（例如，从早上 6 点开始计算）。
    
-   **配置项：** 允许用户设置每日的起始时间点（例如：`00:00`、`06:00` 等）。
    

----------

### 2. 界面与主题设置

#### 2.1 主题配色

允许用户选择应用的主题外观。至少应包含：

-   **浅色模式 (Light Mode)**
    
-   **深色模式 (Dark Mode)**
    

----------

### 3. 数据采集与 AI 分析配置

#### 3.1 采集频率设定

用户可以设置系统活动数据（如当前活动窗口和应用标题）的采集间隔。

-   **配置范围：** 可配置的最小间隔为 1 秒，建议最大间隔为 60 秒。
    

#### 3.2 AI 提供商与接口配置

此区域用于启用和配置外部 AI 驱动的分析功能。

-   **AI 提供商选择：** 允许用户通过下拉菜单选择不同的 AI 服务提供商，例如 `Gemini API`、`OpenAI API` 或 `自定义接口`。
    
-   **API Key / 地址输入：** 提供一个或多个文本输入框，供用户输入所选 AI 服务的 **API Key** 或 **自定义接口地址**。
    
-   **安全要求：** API Key 在本地必须经过加密存储。
    
-   **连接测试：** 需提供一个“测试连接”按钮，验证 API 配置的有效性。
    

#### 3.3 AI 分析频率

用户可以设定 AI 进行数据分析和生成总结的触发时机。

-   **触发选项：**
    
    1.  每个专注周期结束后。
        
    2.  每日一次（例如，在每日数据周期结束时进行总结）。
        
    3.  仅手动触发。
        

----------

### 4. 帮助与支持

#### 4.1 帮助与反馈（采纳）

提供一个区域，其中包含：

-   **版本信息：** 显示当前应用程序的版本号。
    
-   **帮助文档链接：** 指向用户帮助手册或常见问题解答页面。
    
-   **反馈渠道：** 提供用户的反馈渠道，例如电子邮件地址或外部论坛链接。
